# Unity Catalog AI

Welcome to **Unity Catalog AI**, your comprehensive solution for integrating powerful AI functionalities with Unity Catalog. Whether you're a GenAI developer looking to enhance your applications or a Data Engineer seeking robust governance and access control, Unity Catalog AI bridges the gap between advanced AI tools and enterprise-grade data management.

## What is Unity Catalog AI?

**Unity Catalog AI** is a core library designed to seamlessly integrate Unity Catalog with various Generative AI (GenAI) frameworks and tools. It allows you to define, manage, and execute functions within Unity Catalog and utilize them as tools in your GenAI applications. By leveraging Unity Catalog's robust data governance and management capabilities, Unity Catalog AI ensures that your AI tools are secure, well-organized, and easily accessible.

## Key Features

### Function Calling for GenAI

Unity Catalog AI enables you to create and manage functions that can be invoked as tools by AI models. These **UC Functions** encapsulate specific business logic or operations that can be executed on demand, providing AI agents with powerful capabilities to:

- Perform computations and data transformations
- Access and query databases
- Interact with external APIs and services
- Execute custom business logic securely

### Model Registry for ML Governance

Store and manage your machine learning models with Unity Catalog's integrated model registry. The registry provides:

- **Registered Models**: Logical containers for ML models with versioning support
- **Model Versions**: Track different iterations of your models over time
- **Governance**: Apply access controls and audit trails to ML artifacts
- **Integration**: Seamless integration with MLflow for model tracking and deployment

### Framework Integrations

Unity Catalog AI offers native integrations with popular GenAI frameworks:

- **LangChain**: Build sophisticated agent workflows with UC functions as tools
- **LlamaIndex**: Enhance retrieval-augmented generation pipelines
- **OpenAI**: Use UC functions with GPT models for tool calling
- **Anthropic**: Integrate with Claude models for agentic applications
- **CrewAI**: Coordinate multi-agent systems with governed tools
- **AutoGen**: Build conversational AI with secure function access
- **LiteLLM**: Universal API for multiple LLM providers
- **Gemini**: Google's AI models with UC function support

## Key Concepts

### GenAI Tools

**GenAI Tools** are external functions or services that Generative AI models can invoke to perform specific tasks. These tools extend the capabilities of AI models by allowing them to interact with external systems, perform computations, access databases, and more. In Unity Catalog AI, GenAI Tools are UC functions integrated into AI workflows.

### UC Functions

**UC Functions** are functions defined and stored within Unity Catalog. They encapsulate specific business logic or operations that can be executed on demand. Benefits include:

- Centralized management and versioning
- Fine-grained access control
- Audit trails for compliance
- Secure execution environments
- Type validation and schema generation

## Use Cases

### Simplified Agent Authoring

Creating and managing agents in GenAI applications can be complex. Unity Catalog AI simplifies this by:

- **Handling Access Control**: Automatically manages permissions, ensuring only authorized agents can invoke specific functions
- **Parsing and Execution**: Takes care of parsing function inputs and executing them correctly
- **Native Integrations**: Provides per-library integrations for seamless tool usage

### Enhanced Security

Security is paramount when dealing with data and AI functionalities. Unity Catalog AI ensures:

- **Function Access is Secured**: Only authorized users and agents can access and execute UC Functions
- **Granular Permissions**: Define precise permissions at catalog, schema, and function levels
- **Auditability**: Track and audit function usage for compliance and monitoring

### ML Model Governance

Manage the complete lifecycle of machine learning models:

- Register models from training runs
- Version and track model iterations
- Apply governance policies to model artifacts
- Control access to production models
- Integrate with MLflow for seamless workflows
