## Unity Catalog AI Releases

The Unity Catalog AI packages provide AI/ML integration capabilities for Unity Catalog.

### Unity Catalog AI 0.3.1

The 0.3.1 release includes important bug fixes, improved error handling, and enhanced reliability for function execution and client connections.

#### Bug Fixes and Improvements

- **Improved Function Client Defaults**: Fixed default client configuration in `get_uc_function_client` to automatically set up DatabricksFunctionClient when Databricks environment is available
- **Enhanced SQL NULL Handling**: Improved handling of SQL NULL default parameters in both Databricks and OSS clients
- **Better Connection Recovery**: Enhanced retry handling logic for DatabricksFunctionClient with improved session expiration detection
- **Smarter Spark Session Management**: Optimized Spark session creation to only initialize when needed
- **Improved Error Messages**: Added better error handling for Pydantic validation errors
- **Dependency Version Updates**: Set maximum version constraint for `databricks-connect` to `<16.4`
- **Enhanced Documentation**: Added comprehensive development loop tutorial notebook

#### Technical Improvements

- Function execution now returns native Python types instead of string-wrapped values
- Improved function parameter default processing with better JSON parsing
- Enhanced retry mechanisms with exponential backoff
- Better separation of concerns between execution modes
- Streamlined toolkit creation process with automatic client detection

### Unity Catalog AI 0.3.0

The 0.3.0 release includes several usability improvements, a safer function execution mode, and features that expand the functionality of versatile function usage.

#### Major Changes

- **Enhanced Security**: Functions now execute in a safer Process pool by default (using `'sandbox'` `execution_mode`), replacing the original main process implementation. This allows for timeout restrictions, CPU resource limitations, and blocks modules that access system resources. The legacy mode remains available via the `'local'` `execution_mode` parameter.

- **Developer-Friendly Debugging**: A new development-only execution mode for `DatabricksFunctionClient` enables local subprocess execution with runtime protection and timeout controls. While not for production use, this simplifies debugging and rapid iteration without requiring serverless cluster execution. Configure by changing the `execution_mode` from `'serverless'` to `'local'`.

- **Streamlined Integration in Databricks**: When using integration packages (e.g., `unitycatalog-openai`) with a `UCFunctionToolkit` instance, you no longer need to explicitly declare a `DatabricksFunctionClient` within the toolkit interface when a serverless connection is available.

- **Improved Connection Reliability**: Fixed credential refresh issues in the `DatabricksFunctionClient`, including the 24-hour cluster idle time error.

- **Python Callable Access**: New `get_function_as_callable` APIs allow retrieval of UC-registered Python functions as direct Python callables.

- **Function Source Inspection**: Added `get_function_source` APIs to retrieve UC Python function definitions as strings.

- **Warning Suppression**: The persistent Pydantic V2 warning from LangChain API calls has been silenced.

### Unity Catalog AI 0.2.0

The 0.2.0 release brings exciting new functionality for using Unity Catalog for GenAI function execution.

#### Major Changes

- **unitycatalog-autogen** has been overhauled to support the new `0.4.x` version of [AutoGen](https://www.microsoft.com/en-us/research/project/autogen/). Support for earlier versions has been dropped.

- **unitycatalog-gemini** has been released. You can now use Unity Catalog functions as tools when interfacing with Gemini models that support tool calling.

- **unitycatalog-litellm** has been released. You can now interface with the universal common interface of LiteLLM to pass tool usage definitions to integrated providers in LiteLLM that support tool calling.

- **New APIs for function wrapping**. This change eliminates the need for making multiple copies of utility functions that need to be embedded in your Unity Catalog functions.

- **Databricks UC Client** now only supports serverless endpoints using dbconnect for all CRUD and execution APIs.

- **Support for requirements, environment_version and Variant type** have been added to the Databricks client.

### Unity Catalog AI 0.1.0

The initial release of Unity Catalog AI covers the following features and packages.

#### Packages

- **[unitycatalog-ai](https://pypi.org/project/unitycatalog-ai/)** - Core client package for function CRUD operations and execution
- **[unitycatalog-langchain](https://pypi.org/project/unitycatalog-langchain/)** - Integration with LangChain and LangGraph
- **[unitycatalog-llamaindex](https://pypi.org/project/unitycatalog-llamaindex/)** - Integration with LlamaIndex
- **[unitycatalog-openai](https://pypi.org/project/unitycatalog-openai/)** - Integration with OpenAI SDK
- **[unitycatalog-anthropic](https://pypi.org/project/unitycatalog-anthropic/)** - Integration with Anthropic SDK
- **[unitycatalog-crewai](https://pypi.org/project/unitycatalog-crewai/)** - Integration with CrewAI
- **[unitycatalog-autogen](https://pypi.org/project/unitycatalog-autogen/)** - Integration with AutoGen

#### Features

The `unitycatalog-ai` package provides:

- Create Catalogs and Schemas
- List and Delete Unity Catalog Functions
- Create Unity Catalog functions from Python Callable definitions
- Execute a Unity Catalog Function

The integration packages allow for seamless definition of GenAI tools within agent code for GenAI applications through the unified `UCFunctionToolkit` interface.

## Breaking Changes

### 0.3.0 → 0.3.1

- Function execution now returns native Python types instead of string-wrapped values
- Maximum `databricks-connect` version set to `<16.4`

### 0.2.0 → 0.3.0

- Default execution mode changed from `'local'` to `'sandbox'` for enhanced security
- Databricks client now requires serverless endpoints for all operations

### 0.1.0 → 0.2.0

- AutoGen integration requires AutoGen 0.4.x+ (dropped support for earlier versions)
- Databricks client serverless-only support

## Migration Guides

### Upgrading Unity Catalog AI

When upgrading between major versions:

<Steps>
  <Step title="Review Breaking Changes">
    Check the breaking changes section for your target version
  </Step>

  <Step title="Update Dependencies">
    Update your `requirements.txt` or `pyproject.toml`:
    
    ```bash
    pip install --upgrade unitycatalog-ai
    ```
  </Step>

  <Step title="Update Code">
    Modify your code to accommodate any API changes
  </Step>

  <Step title="Test Thoroughly">
    Run your test suite to verify compatibility
  </Step>
</Steps>
