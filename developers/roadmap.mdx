## Overview

This document outlines the roadmap for the Unity Catalog open source project. As always, features may move in/out of milestones pending available resources and priorities.

<Note>
The roadmap is subject to change based on community feedback and priorities. Get involved in discussions on [GitHub](https://github.com/unitycatalog/unitycatalog) or [Slack](https://go.unitycatalog.io/slack).
</Note>

## Version 0.4 Release Priorities

### Storage Management

By more tightly integrating the already released credential and external locations API with the rest of the server internals, the next release will allow for more fine-grained, dynamic, and online management of storage locations and their credentials.

Furthermore, operators can delegate some storage management to the catalog via the managed locations for catalogs and schemas features.

**Key Features:**
- Managed location in catalog
- Managed location in schema
- Enhanced credential management
- External location improvements

### Catalog Managed Commits

Catalog managed commits are the basis for many new and powerful client (Delta) and server side features. Supporting the table scan and commit APIs is a key priority for the upcoming release.

**Key Features:**
- Table scan APIs
- Commit APIs
- Delta Kernel integration
- Enhanced transaction support

### End-to-End OAuth Support

OAuth support is important for cloud users and RBAC, and Unity Catalog plans to support common OAuth flows for authentication.

**Key Features:**
- OAuth client-side support (Spark integration)
- OAuth/OIDC for users
- Enhanced authentication flows
- Token refresh mechanisms

### Sharing

All sharing features are under consideration for future releases:

- **Delta Sharing integration**
- **Shares API**
- **Recipients API**
- **Providers API**

### Federation

All federation features are under consideration for future releases:

- **Connections API**
- **Foreign objects** (catalogs, schemas, tables)
- **Multiple data sources** (JDBC, Iceberg REST, HMS)

### Server Production-Readiness

Planned for v0.5+ to support running as HMS replacement:

- **Monitoring and Telemetry**
- **Database schema upgrades**
- **Change events**

## Legend

- ✅ **Implemented** - Feature is available in released version
- 🟢 **Current Release** - Feature in active development for v0.4
- 🛠️ **In Progress** - Feature partially implemented
- 📋 **Planned** - Feature planned for future release
- 🔮 **Under Consideration** - Feature being evaluated
- ❓ **TBD** - Timeline to be determined
