      Quick Start

    <p className="mt-3 text-base text-slate-600 dark:text-slate-300">
      Follow the shortest path from clone to governed queries, then branch into API, AI, and integration guides.
    </p>
  </div>

  <div className="mt-10">
    <Steps>
      <Step title="Clone and build">
        Clone the repository and build the project locally so the server and CLI binaries are ready.

        ```bash
        git clone https://github.com/unitycatalog/unitycatalog.git
        cd unitycatalog
        build/sbt package
        ```

        If you want a containerized setup instead, use the [Docker guide](/server/docker).
      </Step>

      <Step title="Start the server">
        Launch the default Unity Catalog server and load the sample assets.

        ```bash
        bin/start-uc-server
        ```

        <Note>
          The local server exposes the default API base URL at `http://localhost:8080/api/2.1/unity-catalog`.
        </Note>
      </Step>

      <Step title="Verify with the CLI">
        Confirm the catalog is available, then inspect the sample schema and tables.

        ```bash
        bin/uc catalog list
        bin/uc table list --catalog unity --schema default
        ```

        <Accordion title="What you should see">
          You should see a `unity` catalog plus sample tables such as `marksheet`, `marksheet_uniform`, `numbers`, and `user_countries`.
        </Accordion>

        Continue with the full [CLI overview](/cli/overview) when you are ready to create or inspect more assets.
      </Step>

      <Step title="Connect engines or AI clients">
        Attach a compute engine or SDK client to the running server and start building on top of the catalog.

        ```sql
        CREATE SECRET (
          TYPE UC,
          TOKEN 'not-used',
          ENDPOINT 'http://127.0.0.1:8080',
          AWS_REGION 'us-east-2'
        );

        ATTACH 'unity' AS unity (TYPE UC_CATALOG);
        SELECT * FROM unity.default.numbers;
        ```

        Move next to [integrations](/integrations/overview), the [API reference](/api/overview), or [Unity Catalog AI](/ai/introduction).
      </Step>
    </Steps>

  </div>
</div>
