Unity Catalog is the industry's only universal catalog for data and AI, designed to work seamlessly with multiple compute engines, storage formats, and data platforms.

## Multimodal Interface

Unity Catalog provides a flexible, open interface that supports:

- **Multi-format support** - Extensible support for Delta Lake, Apache Iceberg (via UniForm), Apache Hudi (via UniForm), Apache Parquet, JSON, CSV, and more
- **Multi-engine support** - Open APIs allow data cataloged in Unity Catalog to be accessed by leading compute engines
- **Multimodal assets** - Supports tables, files, functions, and AI models in a unified catalog

## How It Works

Unity Catalog provides open APIs and connectors that enable compute engines to:

1. **Discover metadata** - Browse catalogs, schemas, tables, and other assets
2. **Access data** - Read table metadata and data locations
3. **Vend credentials** - Receive temporary, scoped credentials for secure data access
4. **Enforce governance** - Apply fine-grained access controls and policies

### API Compatibility

Unity Catalog is compatible with multiple catalog APIs:

- Unity Catalog REST API
- Apache Hive Metastore API
- Apache Iceberg REST Catalog API

## Getting Started

Choose your preferred compute engine to get started:

<Steps>
  <Step title="Start Unity Catalog Server">
    Launch the Unity Catalog server on your local machine or deploy it to your infrastructure.
    
    ```bash
    bin/start-uc-server
    ```
  </Step>
  
  <Step title="Choose Your Engine">
    Select the compute engine that best fits your use case and follow the integration guide.
  </Step>
  
  <Step title="Configure Connection">
    Set up the connection to your Unity Catalog server using the engine-specific connector or configuration.
  </Step>
  
  <Step title="Query Your Data">
    Start querying tables and accessing data through Unity Catalog's unified interface.
  </Step>
</Steps>
