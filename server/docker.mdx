Unity Catalog provides official Docker images and Docker Compose configurations for easy deployment and development.

## Quick Start

The fastest way to get Unity Catalog running is with Docker Compose.

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed
- At least 4GB of available memory allocated to Docker

### Start Unity Catalog

From the Unity Catalog project root directory:

```bash
docker compose up -d
```

This command starts:
- Unity Catalog server on `http://localhost:8080`
- Unity Catalog UI on `http://localhost:3000`

<Note>
The `-d` flag runs containers in detached mode (background). Omit it to see logs in your terminal.
</Note>

### Verify the Deployment

Check that both services are running:

```bash
docker compose ps
```

You should see:

```console
NAME                       IMAGE                              STATUS
unitycatalog-server-1      unitycatalog/unitycatalog:latest   Up
unitycatalog-ui-1          unitycatalog-ui:latest             Up
```

## Using the CLI

Access the Unity Catalog CLI inside the running container:

```bash
docker exec -it unitycatalog-server-1 /bin/bash
```

Once inside the container, use the CLI:

```bash
# List tables
bin/uc table list --catalog unity --schema default

# Get table metadata
bin/uc table get --full_name unity.default.numbers

# List catalogs
bin/uc catalog list
```

To exit the container shell:

```bash
exit
```

## Accessing the Services

### Unity Catalog Server

The server API is accessible at:

```
http://localhost:8080
```

Test the connection:

```bash
curl http://localhost:8080/api/2.1/unity-catalog/catalogs
```

### Unity Catalog UI

Open your browser and navigate to:

```
http://localhost:3000
```

The UI provides a visual interface to browse catalogs, schemas, tables, and other Unity Catalog assets.

## Docker Compose Configuration

The `compose.yaml` file in the project root defines the services:

```yaml compose.yaml
name: unitycatalog

services:

  server:
    image: unitycatalog/unitycatalog:latest
    ports:
      - "8080:8080"
    volumes:
      - type: bind
        source: ./etc/conf
        target: /opt/unitycatalog/etc/conf
      - type: volume
        source: unitycatalog_data
        target: /opt/unitycatalog/etc/data
  
  ui:
    build:
      context: ui/
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    depends_on:
      - server

volumes:
  # Persist docker volume across container restarts
  unitycatalog_data:
```

### Service Breakdown

<ResponseField name="server" type="service">
  Unity Catalog server container:
  - Uses official `unitycatalog/unitycatalog:latest` image
  - Exposes port `8080`
  - Mounts configuration and data directories
</ResponseField>

<ResponseField name="ui" type="service">
  Unity Catalog UI container:
  - Built from `ui/Dockerfile`
  - Exposes port `3000`
  - Depends on server being available
</ResponseField>

## Volume Mounts

The Docker Compose setup uses two types of volume mounts:

### Configuration Bind Mount

```yaml
- type: bind
  source: ./etc/conf
  target: /opt/unitycatalog/etc/conf
```

This bind mount allows you to edit configuration files on your host machine, and changes are immediately reflected in the container.

**Configuration files:**
- `etc/conf/server.properties` - Server configuration
- `etc/conf/hibernate.properties` - Database settings
- `etc/conf/server.log4j2.properties` - Logging configuration

### Data Persistent Volume

```yaml
- type: volume
  source: unitycatalog_data
  target: /opt/unitycatalog/etc/data
```

This named volume persists data across container restarts, ensuring your catalogs, tables, and metadata aren't lost when containers are stopped or removed.

## Configuring the Server

### Edit Configuration Files

Since configuration files are bind-mounted, you can edit them directly on your host:

```bash
# Edit server configuration
vim etc/conf/server.properties

# Edit database configuration
vim etc/conf/hibernate.properties
```

### Restart After Configuration Changes

After modifying configuration files, restart the services:

```bash
docker compose restart server
```

### Example: Enable Authentication

Edit `etc/conf/server.properties`:

```properties
server.authorization=enable
server.authorization-url=https://accounts.google.com/o/oauth2/auth
server.token-url=https://oauth2.googleapis.com/token
server.client-id=your-client-id
server.client-secret=your-client-secret
```

Restart the server:

```bash
docker compose restart server
```

## Custom Docker Compose Setup

You can customize the Docker Compose configuration for your needs.

### Change Ports

```yaml
services:
  server:
    ports:
      - "9090:8080"  # Map host port 9090 to container port 8080
  
  ui:
    ports:
      - "4000:3000"  # Map host port 4000 to container port 3000
```

### Use External Database

Add a PostgreSQL service:

```yaml
services:
  server:
    image: unitycatalog/unitycatalog:latest
    ports:
      - "8080:8080"
    depends_on:
      - postgres
    volumes:
      - type: bind
        source: ./etc/conf
        target: /opt/unitycatalog/etc/conf
  
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: ucdb
      POSTGRES_USER: uc_user
      POSTGRES_PASSWORD: uc_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
  
  ui:
    build:
      context: ui/
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    depends_on:
      - server

volumes:
  postgres_data:
```

Update `etc/conf/hibernate.properties`:

```properties
hibernate.connection.driver_class=org.postgresql.Driver
hibernate.connection.url=jdbc:postgresql://postgres:5432/ucdb
hibernate.connection.user=uc_user
hibernate.connection.password=uc_password
```

### Environment Variables

Pass environment variables to containers:

```yaml
services:
  server:
    image: unitycatalog/unitycatalog:latest
    environment:
      - JAVA_OPTS=-Xmx4G -Xms2G
      - AWS_REGION=us-east-1
    env_file:
      - .env.production
```

## Building Custom Images

### Build Server Image

The project includes a Dockerfile for building the server image:

```dockerfile Dockerfile
# syntax=docker.io/docker/dockerfile:1.7-labs
ARG HOME="/home/unitycatalog"
ARG ALPINE_VERSION="3.20"

# Build stage
FROM amazoncorretto:17-alpine${ALPINE_VERSION}-jdk as base

ARG HOME
ENV HOME=$HOME

WORKDIR $HOME

COPY --parents dev/ build/ project/ examples/ server/ api/ clients/ version.sbt build.sbt ./

RUN apk add --no-cache bash && ./build/sbt -info clean package

# Runtime stage
FROM alpine:${ALPINE_VERSION} as runtime

ARG JAVA_HOME="/usr/lib/jvm/default-jvm"
ARG USER="unitycatalog"
ARG HOME

# Copy Java from base
COPY --from=base $JAVA_HOME $JAVA_HOME

ENV HOME=$HOME \
    JAVA_HOME=$JAVA_HOME \
    PATH="${JAVA_HOME}/bin:${PATH}"

# Copy build artifacts
COPY --from=base --parents \
    $HOME/examples/ \
    $HOME/server/ \
    $HOME/api/ \
    $HOME/clients/ \
    $HOME/target/ \
    $HOME/.cache/ \
    /

# Create service user
RUN apk add --no-cache bash && \
    addgroup -S $USER && \
    adduser -S -G $USER $USER && \
    chmod -R 550 $HOME && \
    mkdir -p $HOME/etc/ && \
    chmod -R 770 $HOME/etc/ && \
    chown -R $USER:$USER $HOME

USER $USER

# Copy remaining directories
COPY --chown=$USER:$USER --parents bin/ etc/ $HOME/

WORKDIR $HOME

CMD ["./bin/start-uc-server"]
```

Build the image:

```bash
docker build -t my-unitycatalog:latest .
```

Use your custom image in `compose.yaml`:

```yaml
services:
  server:
    image: my-unitycatalog:latest
    # ... rest of configuration
```

## Managing Containers

### View Logs

```bash
# View all logs
docker compose logs

# Follow logs in real-time
docker compose logs -f

# View server logs only
docker compose logs server

# View UI logs only
docker compose logs ui
```

### Stop Services

```bash
# Stop containers (data persists)
docker compose stop

# Stop and remove containers (data persists in volumes)
docker compose down
```

### Restart Services

```bash
# Restart all services
docker compose restart

# Restart only the server
docker compose restart server
```

### Remove Everything

<Warning>
This command removes all containers, networks, and volumes. All data will be lost.
</Warning>

```bash
docker compose down --volumes --remove-orphans
```

## Connecting External Clients

Clients running on your host machine can connect to Unity Catalog:

### DuckDB

```sql
install uc_catalog from core_nightly;
load uc_catalog;
install delta;
load delta;

CREATE SECRET (
  TYPE UC,
  TOKEN 'not-used',
  ENDPOINT 'http://localhost:8080',
  AWS_REGION 'us-east-2'
);

ATTACH 'unity' AS unity (TYPE UC_CATALOG);
SHOW ALL TABLES;
```

### Spark

```bash
spark-sql --name "uc-test" \
  --master "local[*]" \
  --packages "io.delta:delta-spark_2.13:4.0.0,io.unitycatalog:unitycatalog-spark_2.13:0.3.0" \
  --conf "spark.sql.catalog.unity=io.unitycatalog.spark.UCSingleCatalog" \
  --conf "spark.sql.catalog.unity.uri=http://localhost:8080" \
  --conf "spark.sql.defaultCatalog=unity"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container fails to start">
    Check logs for errors:
    ```bash
    docker compose logs server
    ```
    
    Ensure sufficient resources:
    ```bash
    docker system info | grep Memory
    ```
  </Accordion>

  <Accordion title="Port already in use">
    Change the port mapping in `compose.yaml`:
    ```yaml
    ports:
      - "8081:8080"  # Use port 8081 on host
    ```
  </Accordion>

  <Accordion title="Configuration changes not applied">
    Restart the container after modifying configuration:
    ```bash
    docker compose restart server
    ```
  </Accordion>

  <Accordion title="Cannot connect from host machine">
    Verify the container is running and ports are exposed:
    ```bash
    docker compose ps
    curl http://localhost:8080/api/2.1/unity-catalog/catalogs
    ```
  </Accordion>
</AccordionGroup>
